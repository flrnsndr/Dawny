Git Solo Cheat Sheet

🆘 Oops! Auf Main committet

Du hast Änderungen im main gespeichert, die in einen neuen Branch sollen:

1. Code retten & Main zurücksetzen git branch feature-name git reset --hard HEAD~1 git checkout feature-name
Erklärung: Erstellt den neuen Branch (mit dem Code), setzt Main einen Schritt zurück und springt dann in den neuen Branch.

🚀 Daily Loop

1. Update Main git checkout main git pull
2. Neuer Branch git checkout -b feature-name
3. Sichern & Push git add . git commit -m "update" git push -u origin HEAD

🔍 Übersicht

Status & Branch-Check git status git branch -av
Grafischer Verlauf git log --oneline --graph --all -n 10

🧹 Aufräumen (nach Merge)

1. Sync & Prune git checkout main git pull git fetch -p
2. Branch lokal löschen git branch -d feature-name
Alle lokalen Branches außer main (nur gemergt, -d) git checkout main git branch --format='%(refname:short)' | grep -vx 'main' | xargs git branch -d
Alle lokalen Branches außer main (inkl. ungemergt, -D) git checkout main git branch --format='%(refname:short)' | grep -vx 'main' | xargs git branch -D

🔄 Branch aktualisieren

Wenn Main sich geändert hat, während du arbeitest:

Variante A: Rebase (Sauber) git checkout main git pull git checkout - git rebase main
Variante B: Merge (Einfach) git merge main

🛠️ Rettung

Alles im Branch verwerfen git reset --hard HEAD
Reflog (Letzte Rettung) git reflog
